# 红利低波打分器使用说明

从 0 到 1 推荐先读 [README.md](./README.md)。

本文档说明如何在红利低波产品线上使用评分器、策略和回测验证路径。

## 1. 当前目录与角色

当前主实现都收敛到 `skyeye/products/dividend_low_vol/`：

- `scorer/`
  估值打分器、缓存同步、参数审计。
- `strategies/history_aware/`
  当前基线策略，包含 `strategy.py`、`spec.yaml`、`profiles/` 和 `logic/`。
- `mod/rqalpha_mod_dividend_scorer/`
  给 RQAlpha 暴露 `get_dividend_score()` 的产品级 mod。

后续研发建议固定走三条验证路径：

- `rqalpha run --plot-save`
  看具体区间净值、回撤、基准对比和图片。
- `python -m skyeye.evaluation.rolling_score.cli`
  看跨年份稳定性，防止只在单一区间有效。
- `uv run pytest`
  看 scorer、strategy、evaluation 和基础单元测试有没有回归。

## 2. 环境准备

```bash
uv venv --python 3.13
uv sync --python 3.13 --extra dividend-scorer
uv run rqalpha download-bundle
```

如果仓库根目录已经有 `.venv/` 且依赖已经装好，可以跳过 `uv venv`。
如果只是更新依赖，直接执行 `uv sync --python 3.13 --extra dividend-scorer` 即可。

默认路径：

- SQLite 缓存：`~/.rqalpha/dividend_scorer/cache.db`
- bundle：`~/.rqalpha/bundle`

## 3. 同步评分缓存

```bash
uv run python -m skyeye.products.dividend_low_vol.scorer.main \
  --sync-only \
  --end-date "$(date +%F)"
```

说明：

- 首次同步会从 `2020-01-01` 全量建库。
- 后续默认按“上次成功日期往前回溯 10 个交易日 -> 今天”做增量同步。
- 如果某个数据源缓存已经完整覆盖请求区间，会命中 checkpoint 并直接跳过。

## 4. 直接查看评分

查看当前最新可用日期：

```bash
uv run python -m skyeye.products.dividend_low_vol.scorer.main
```

查看 JSON：

```bash
uv run python -m skyeye.products.dividend_low_vol.scorer.main --json
```

查看指定日期：

```bash
uv run python -m skyeye.products.dividend_low_vol.scorer.main --date "$(date +%F)"
```

常用字段：

- `total_score`
- `score_percentile`
- `confidence`
- `features`
- `confidence_modifiers`
- `model_meta`
- `data_freshness`

## 5. 在 RQAlpha 策略里使用

只要启用 `dividend_scorer` mod，策略里就可以直接调用 `get_dividend_score()`：

```python
from rqalpha.apis import *


def handle_bar(context, bar_dict):
    score = get_dividend_score()
    if not score or score.get("error"):
        return
```

当前基线策略入口：

- [strategy.py](/home/tiger/rqalpha/skyeye/products/dividend_low_vol/strategies/history_aware/strategy.py)

它的核心约定：

- 主信号来自最近 `5/15` 期 `score_percentile` 的快慢均值和趋势，不是单日阈值开平仓。
- 单日分位只描述当前位置，不直接等于交易动作。
- `heat_override`、`confidence_slowdown`、`high_reentry_guard` 是显式风险控制模块。
- `spec.yaml` 固定记录策略假设、适用环境和主规则，避免结论只写在代码注释里。

## 6. 跑单区间回测并导出图片

```bash
env UV_CACHE_DIR=/tmp/uv-cache MPLCONFIGDIR=/tmp/mplconfig uv run rqalpha run \
  -f skyeye/products/dividend_low_vol/strategies/history_aware/strategy.py \
  -s 2024-05-03 \
  -e 2025-04-20 \
  --account stock 100000 \
  -fq 1d \
  -d ~/.rqalpha/bundle \
  -bm 512890.XSHG \
  --plot-save dividend_price_2405_2504.png \
  -mc dividend_scorer.enabled true \
  -mc dividend_scorer.db_path ~/.rqalpha/dividend_scorer/cache.db
```

这条命令用来直接回答“这段时间策略有没有跑赢红利低波 ETF 自身”。

如果你要切换同一策略的参数档位，可以继续传：

```bash
-mc extra.strategy_profile aggressive
```

对应 profile 会从：

- [profiles](/home/tiger/rqalpha/skyeye/products/dividend_low_vol/strategies/history_aware/profiles)

里加载。

## 7. 跑滚动窗口打分

```bash
uv run python -m skyeye.evaluation.rolling_score.cli \
  skyeye/products/dividend_low_vol/strategies/history_aware/strategy.py \
  --mod dividend_scorer \
  -mc dividend_scorer.db_path ~/.rqalpha/dividend_scorer/cache.db \
  --log high
```

这条命令会额外输出策略卡片，卡片内容来自：

- [spec.yaml](/home/tiger/rqalpha/skyeye/products/dividend_low_vol/strategies/history_aware/spec.yaml)

所以后续策略演进时，不应该再把关键假设、适用环境和风控描述只塞进单文件代码里。

## 8. 参数审计

如果你改了打分器权重、阈值或 blend 逻辑，先跑参数审计：

```bash
uv run python -m skyeye.products.dividend_low_vol.scorer.parameter_audit \
  --db-path ~/.rqalpha/dividend_scorer/cache.db
```

这一步用来看低分组是否持续优于高分组，以及不同年份里结论是否退化。

## 9. 跑基础单元测试

```bash
env UV_CACHE_DIR=/tmp/uv-cache PYTHONPATH="$PWD" MPLCONFIGDIR=/tmp/mplconfig uv run pytest -q \
  tests/dividend_scorer \
  tests/unittest \
  tests/evaluation \
  tests/products
```

建议每次改完 scorer、strategy、rolling_score 入口后，至少跑这一组基础测试。
如果你要跑 `tests/` 全量，先补齐 `TA-Lib`，否则 `tests/integration_tests/test_backtest_results/` 会在收集阶段失败。

## 10. 当前的可视化验证路径

当前不再保留独立的 `plot_score_vs_price.py`。
如果你要看单区间真实净值和基准对比，直接使用上面的 `rqalpha run --plot-save`。
如果你要看滚动窗口里的买卖点图，使用 `python -m skyeye.evaluation.rolling_score.cli -w 37 --plot`。

## 11. 全局配置入口

命令行临时启用 mod 往往已经够用。如果你确实要改全局配置，当前 `dividend_scorer` 的 lib 路径是：

```yaml
mod:
  dividend_scorer:
    enabled: false
    lib: skyeye.products.dividend_low_vol.mod.rqalpha_mod_dividend_scorer
```

对应文件：

- [mod_config.yml](/home/tiger/rqalpha/rqalpha/mod_config.yml)
