# 策略回测打分器

主入口：`skyeye/strategy_scorer.py`

从 0 到 1 的推荐路径见 [README.md](./README.md)。

本文档说明如何使用 `skyeye/strategy_scorer.py` 对策略做滚动窗口回测打分，包括：

- 环境前提和适用范围
- 最常用命令
- 各参数的实际含义
- 输出结果怎么解读
- 常见误区和排障建议

## 1. 它能做什么

`skyeye/strategy_scorer.py` 会对一个策略文件做 37 个滚动窗口回测，然后给出：

- 策略综合得分
- 策略稳定得分
- 核心指标摘要
- 牛市 / 震荡 / 熊市三类市场环境下的表现
- 逐笔交易日志
- 可选的买卖点 PNG 图表

它和单次 `rqalpha run --plot-save` 是互补关系：

- `skyeye/strategy_scorer.py`：看多窗口稳定性和跨市场环境表现
- `rqalpha run --plot-save`：看某一个具体区间的真实净值、基准对比和导出图片

评分逻辑不是单次回测看收益率，而是：

1. 固定 37 个滚动窗口
2. 每个窗口基于 13 个指标单独打分
3. 把窗口结果投影到季度网格，避免重叠窗口重复计分
4. 对最近季度给予更高权重
5. 额外计算稳定性分和市场环境适应性

## 2. 适用范围

当前版本更适合股票 / ETF / LOF 一类按日频运行的策略，不是任意资产类别的通用评分框架。脚本内部有几个固定前提：

- 账户类型固定为 `stock`
- 频率固定为 `1d`
- 基准固定为 `000300.XSHG`
- 滚动窗口固定从 `2016-02-01` 开始，到第 37 个窗口结束于 `2026-01-31`
- bundle 路径固定为 `~/.rqalpha/bundle`

如果你的策略依赖：

- 期货账户
- 分钟级别数据
- 自定义 benchmark
- 自定义 bundle 路径

那这份脚本当前不能直接覆盖，需要先改代码。

## 3. 环境准备

### 3.1 安装项目依赖

推荐在仓库根目录统一用 `uv`：

```bash
uv venv --python 3.13
uv sync --python 3.13
```

之后统一用：

```bash
uv run python skyeye/strategy_scorer.py --help
```

如果你已经在虚拟环境里安装过项目，也可以直接运行：

```bash
python skyeye/strategy_scorer.py --help
```

项目基础依赖里已经包含：

- `numpy`
- `pandas`
- `h5py`
- `matplotlib`

所以 `--plot` 不需要额外再装图形库。

如果你要给依赖 `dividend_scorer` 的策略打分，建议直接按 [README.md](./README.md)
里的完整环境准备方式走一遍，而不是只停在基础依赖。

### 3.2 准备 bundle

脚本会直接读取 `~/.rqalpha/bundle` 下的数据文件。如果本地还没有 bundle，先执行：

```bash
uv run rqalpha download-bundle
```

至少要有这些文件中的相关数据：

- `instruments.pk`
- `stocks.h5`
- `funds.h5`
- `indexes.h5`

## 4. 策略文件要求

被评分的策略文件需要是标准 RQAlpha 策略，至少应包含诸如：

- `init(context)`
- `handle_bar(context, bar_dict)`

最简单的验证方式就是先确认该策略本身能正常被 RQAlpha 跑起来。

仓库里可直接拿来测试的示例：

- `rqalpha/examples/demo_strategy.py`
- `rqalpha/examples/golden_cross.py`
- `skyeye/examples/dividend_low_vol_score_strategy_history_aware.py`

## 5. 快速开始

下面的命令默认都在仓库根目录执行。

### 5.1 对一个策略做完整打分

```bash
uv run python skyeye/strategy_scorer.py rqalpha/examples/demo_strategy.py
```

这是最常用的入口。默认行为：

- 初始资金 `1000000`
- 跑全部 37 个窗口
- 交易日志展示最近 1 个窗口

### 5.2 指定初始资金

```bash
uv run python skyeye/strategy_scorer.py \
  rqalpha/examples/demo_strategy.py \
  --cash 200000
```

### 5.3 只跑部分窗口

适合快速调试，不适合拿来做完整结论。

```bash
uv run python skyeye/strategy_scorer.py rqalpha/examples/demo_strategy.py -w 37
uv run python skyeye/strategy_scorer.py rqalpha/examples/demo_strategy.py -w 35-37
uv run python skyeye/strategy_scorer.py rqalpha/examples/demo_strategy.py -w 1,10,20,37
```

支持的写法：

- `37`
- `35-37`
- `1,10,37`

注意：

- 窗口编号范围只能是 `1-37`
- 成功窗口少于 5 个时，脚本不会计算综合分，只会输出单窗口分数和交易日志

### 5.4 查看更多交易日志

```bash
uv run python skyeye/strategy_scorer.py \
  rqalpha/examples/demo_strategy.py \
  --log mid
```

`--log` 可选：

- `low`：最近 1 个窗口
- `mid`：最近 4 个窗口
- `high`：全部成功窗口

### 5.5 画买卖点图

```bash
uv run python skyeye/strategy_scorer.py \
  rqalpha/examples/demo_strategy.py \
  -w 37 \
  --plot
```

图表行为：

- 不会弹窗
- 会直接保存到当前工作目录
- 文件名格式为 `trade_w窗口号_证券代码.png`

例如：

- `trade_w37_000001_XSHE.png`

### 5.6 搜索证券代码或名称

`--search` 模式不会跑回测，只查 bundle 里的 Active 证券。

```bash
uv run python skyeye/strategy_scorer.py --search "平安"
uv run python skyeye/strategy_scorer.py -s "510300"
uv run python skyeye/strategy_scorer.py -s "^600[0-9]{3}"
```

搜索规则：

- 正则匹配
- 同时匹配 `order_book_id` 和 `symbol`
- 仅显示状态为 `Active` 的股票 / ETF / LOF / 指数

## 6. 常用参数说明

| 参数 | 说明 |
| --- | --- |
| `strategy_file` | 要评分的策略文件路径 |
| `--cash` | 初始资金，默认 `1000000` |
| `--window`, `-w` | 指定窗口编号，支持单个、区间、逗号列表 |
| `--log` | 交易日志级别：`low` / `mid` / `high` |
| `--plot` | 生成价格走势和买卖点图，保存为 PNG |
| `--mod` | 启用额外 mod，例如 `--mod dividend_scorer` |
| `--mod-config`, `-mc` | 传入 mod 配置，写法为 `-mc <mod>.<field> <value>`，可重复传入 |
| `--search`, `-s` | 搜索证券，不跑回测 |
| `--help`, `-h` | 查看帮助 |
| `--eg` | 查看脚本内置完整示例 |

## 7. 与 mod 联用

如果你的策略依赖额外 mod，通常要同时做两件事：

- 用 `--mod` 打开 mod
- 用 `-mc KEY VALUE` 传入需要覆盖的 mod 配置；该参数可重复传入

红利低波策略最常见的写法：

```bash
uv run python skyeye/strategy_scorer.py \
  skyeye/examples/dividend_low_vol_score_strategy_history_aware.py \
  --mod dividend_scorer \
  -mc dividend_scorer.db_path ~/.rqalpha/dividend_scorer/cache.db \
  --log high
```

如果你还想临时覆盖更多配置，例如权重融合参数：

```bash
uv run python skyeye/strategy_scorer.py \
  skyeye/examples/dividend_low_vol_score_strategy_history_aware.py \
  --mod dividend_scorer \
  -mc dividend_scorer.db_path ~/.rqalpha/dividend_scorer/cache.db \
  -mc dividend_scorer.prior_blend 0.7 \
  -mc dividend_scorer.dynamic_diagnostic false \
  --log high
```

如果要启用多个 mod，本身也支持：

```bash
uv run python skyeye/strategy_scorer.py my_strategy.py --mod mod_a mod_b mod_c
```

注意：

- `--mod` 只负责加载 mod，不会替你决定该用哪个缓存库或实验参数
- `-mc` 是正式支持的入口，适合显式传入 `db_path`、`prior_blend` 等配置
- 对 `dividend_scorer` 来说，建议把 `db_path` 明确写出来，避免误用旧缓存或空库
- 如果缓存还没准备好，先看 [红利低波打分器使用说明.md](./红利低波打分器使用说明.md)

## 8. 稀疏窗口是什么，为什么要关心

滚动窗口固定从 `2016-02-01` 开始，这对晚上市 ETF、长期空仓策略、很久以后才第一次建仓的策略很容易造成误判。

脚本会把下面这些情况标记为“样本不足”：

- 窗口内无持仓日
- 首次建仓比窗口开始晚超过 60 天
- 持仓交易日少于 60 天
- 持仓覆盖率低于 25%

这类窗口常见问题：

- 年化收益被极少数交易日放大
- 夏普 / Sortino 失真
- 综合分被很不公平地拉低或拉高

如果你是在评估晚上市标的，优先建议让策略自己声明更合理的起始窗口。
例如红利低波 `history_aware` 策略会通过 `STRATEGY_SCORER_START_DATE`
把默认滚动窗口收敛到 `2020-02-01` 之后：

```bash
uv run python skyeye/strategy_scorer.py \
  skyeye/examples/dividend_low_vol_score_strategy_history_aware.py \
  --mod dividend_scorer \
  -mc dividend_scorer.db_path ~/.rqalpha/dividend_scorer/cache.db \
  --log high
```

注意：

- 仍然被标记为“样本不足”的窗口，说明策略本身建仓太晚或持仓过稀
- 这种情况下更适合先修策略，而不是靠跳过窗口来美化综合分

## 9. 输出结果怎么解读

### 9.1 策略综合得分

这是主分数。它来自：

- 37 个滚动窗口
- 13 个指标评分
- 季度投影
- 时间衰减加权

分数可以是负数，不是 0 到 100 的封顶分。

### 9.2 策略稳定得分

这是 `0-100` 分，用来回答“这个策略是否稳定，而不是只在少数时间段好看”。

稳定分综合考虑：

- 各季度分数的波动程度
- 最差季度有多差
- 连续低分季度有多长

### 9.3 核心指标

摘要里会显示四个核心指标：

- 年化收益率
- 最大回撤
- 夏普率
- 胜率

这些不是单窗口值，而是按季度投影后再做时间衰减加权得到的代表值。

### 9.4 市场环境

脚本会按沪深 300 的季度涨跌幅把季度划成三类：

- 牛市：季度收益率 `> 8%`
- 熊市：季度收益率 `< -8%`
- 震荡：其余情况

然后分别给出这三类环境下的平均得分。

### 9.5 评级

输出里的评级阈值是：

- `E`：`>= 60`
- `M+`：`>= 30`
- `M`：`>= 0`
- `M-`：`>= -20`
- `I`：`< -20`

### 9.6 过拟合风险提示

当前版本会额外检查两类迹象：

- 前半段季度均分显著高于后半段，说明近期可能失效
- 最佳季度远高于中位数，说明收益可能集中在少数时段

出现提示不代表一定过拟合，但说明这个策略不适合只看单个漂亮区间。

## 10. 交易日志怎么看

交易日志是逐笔展开后的流水，不只是简单打印原始成交表。

它会展示：

- 买入 / 卖出方向
- 成交价格、数量、金额
- 每笔卖出的已实现盈亏
- 当前持仓数量
- 组合级累计已实现盈亏

有两个口径要特别注意：

- `卖出胜率` 只统计已经平仓的卖出
- 它不包含期末仍在持有仓位的浮盈浮亏

另外因为滚动窗口本身互相重叠：

- 同一个真实卖点可能会在多个窗口重复出现
- 所以“卖出笔数”可能明显大于“实际卖点日数量”

脚本在输出里会显式提醒这件事。

## 11. 推荐使用姿势

### 11.1 先用最新窗口检查策略是否跑得通

```bash
uv run python skyeye/strategy_scorer.py my_strategy.py -w 37 --log high
```

先确认：

- 策略没有运行时异常
- 交易方向符合预期
- 交易日志里没有明显的仓位错误

### 11.2 再跑最近几个窗口看一致性

```bash
uv run python skyeye/strategy_scorer.py my_strategy.py -w 33-37 --log high
```

这一步主要看：

- 不同年份附近是否都能工作
- 胜率、回撤、年化是否极度飘忽

### 11.3 最后再跑完整 37 窗口做正式结论

```bash
uv run python skyeye/strategy_scorer.py my_strategy.py --log mid
```

如果是晚上市标的，优先让策略自己声明 `STRATEGY_SCORER_START_DATE`，而不是依赖额外跳窗参数。

## 12. 常见问题

### 12.1 为什么只跑了几个窗口就没有综合分

因为当前实现要求成功窗口至少 5 个。少于 5 个时，只输出单窗口得分和交易日志，不做综合评分。

常见触发原因：

- 你手动用了 `-w 35-37`
- 多个窗口运行失败

### 12.2 为什么前几个窗口分数特别低

常见原因是标的上市太晚，或者策略很久以后才第一次建仓。

例如：

- 窗口从 `2016-02-01` 就开始滚
- 但你的 ETF 可能是 `2019` 年以后才上市

这时建议重点看：

- 输出里的“样本不足”提示
- 策略是否已经声明合适的 `STRATEGY_SCORER_START_DATE`

### 12.3 `--plot` 为什么没有弹窗

这是当前脚本的设计。它使用非交互后端，只保存 PNG，不弹出 GUI 窗口。

### 12.4 `--search` 为什么搜不到结果

先检查：

- 本地是否已下载 bundle
- `~/.rqalpha/bundle/instruments.pk` 是否存在
- 正则写法是否正确
- 你搜的是不是已经退市或非 Active 标的

### 12.5 运行时报 bundle 文件缺失

脚本会直接依赖 `~/.rqalpha/bundle`。先执行：

```bash
uv run rqalpha download-bundle
```

然后再重试。
