# 策略回测打分器

主入口：`python -m skyeye.evaluation.rolling_score.cli`

从 0 到 1 的推荐路径见 [README.md](./README.md)。

## 1. 它解决什么问题

`rqalpha run --plot-save` 只能回答某一个区间里策略净值怎么样。滚动窗口打分器回答的是：

- 策略是不是只在单一区间碰巧有效
- 收益、回撤、夏普、胜率在不同年份是否稳定
- 牛市 / 震荡 / 熊市里，策略表现分布如何

输出包括：

- 策略卡片
- 策略综合得分
- 策略稳定得分
- 核心指标摘要
- 牛市 / 震荡 / 熊市环境分数
- 逐笔交易日志
- 可选买卖点 PNG 图

## 2. 运行前提

- 适合股票 / ETF / LOF 这类按日频运行的策略
- 账户类型固定为 `stock`
- 频率固定为 `1d`
- bundle 默认读取 `~/.rqalpha/bundle`

如果策略目录已经在产品线下注册了 `spec.yaml`，运行时会自动打印策略卡片。
如果策略文件定义了 `ROLLING_SCORE_START_DATE`，打分器会自动跳过更早的无效窗口。

## 3. 环境准备

```bash
uv venv --python 3.13
uv sync --python 3.13 --extra dividend-scorer
uv run rqalpha download-bundle
```

如果仓库根目录已经有 `.venv/` 且依赖已经装好，可以跳过 `uv venv`。
如果只是更新依赖，直接执行 `uv sync --python 3.13 --extra dividend-scorer` 即可。

## 4. 常用命令

普通策略快速打分：

```bash
uv run python -m skyeye.evaluation.rolling_score.cli \
  rqalpha/examples/demo_strategy.py
```

红利低波历史感知策略打分：

```bash
uv run python -m skyeye.evaluation.rolling_score.cli \
  skyeye/products/dividend_low_vol/strategies/history_aware/strategy.py \
  --mod dividend_scorer \
  -mc dividend_scorer.db_path ~/.rqalpha/dividend_scorer/cache.db
```

只跑最近窗口：

```bash
uv run python -m skyeye.evaluation.rolling_score.cli \
  skyeye/products/dividend_low_vol/strategies/history_aware/strategy.py \
  -w 37 \
  --mod dividend_scorer \
  -mc dividend_scorer.db_path ~/.rqalpha/dividend_scorer/cache.db \
  --log high
```

画买卖点图：

```bash
uv run python -m skyeye.evaluation.rolling_score.cli \
  rqalpha/examples/demo_strategy.py \
  -w 37 \
  --plot
```

搜索证券：

```bash
uv run python -m skyeye.evaluation.rolling_score.cli --search "平安"
uv run python -m skyeye.evaluation.rolling_score.cli -s "510300"
uv run python -m skyeye.evaluation.rolling_score.cli -s "^600[0-9]{3}"
```

## 5. 关键参数

| 参数 | 说明 |
| --- | --- |
| `strategy_file` | 要评分的策略文件路径 |
| `--cash` | 初始资金，默认 `1000000` |
| `--window`, `-w` | 指定窗口编号，支持 `37`、`35-37`、`1,10,37` |
| `--log` | 交易日志级别：`low` / `mid` / `high` |
| `--plot` | 生成价格走势和买卖点图 |
| `--mod` | 启用额外 mod，例如 `--mod dividend_scorer` |
| `-mc`, `--mod-config` | 传入 mod 配置，写法为 `-mc <mod>.<field> <value>` |
| `--search`, `-s` | 搜索证券，不跑回测 |
| `--eg` | 查看内置完整示例 |

## 6. 如何理解结果

- 综合得分：多窗口 13 项指标加权后的总分，用来看整体强弱。
- 稳定得分：看波动、最差季度和连续低分情况，用来防止只在少数窗口有效。
- 市场环境：按沪深 300 季度涨跌分成牛市 / 震荡 / 熊市，观察适应性。
- 交易日志：验证仓位变化、成交节奏和卖出盈亏是否符合策略假设。

## 7. 和单区间原生回测怎么配合

- `rqalpha run --plot-save`
  用来回答某一段时间策略净值有没有跑赢基准，并生成图片。
- `python -m skyeye.evaluation.rolling_score.cli`
  用来回答策略跨年份是否稳定，是否只是某一段时间碰巧有效。

对于红利低波策略，建议先用原生回测看 `512890.XSHG` 的单区间真实净值，再用滚动打分看跨年份稳定性。

## 8. 与 `dividend_scorer` mod 联用

如果策略依赖评分器，通常要同时传两类参数：

- `--mod dividend_scorer`
- `-mc dividend_scorer.<field> <value>`

例如：

```bash
uv run python -m skyeye.evaluation.rolling_score.cli \
  skyeye/products/dividend_low_vol/strategies/history_aware/strategy.py \
  --mod dividend_scorer \
  -mc dividend_scorer.db_path ~/.rqalpha/dividend_scorer/cache.db \
  -mc dividend_scorer.prior_blend 0.7
```

如果你需要在 CLI 里切换同一策略的参数档位，可以直接传：

```bash
-mc extra.strategy_profile aggressive
```

对应 profile 会从 `strategies/<strategy_id>/profiles/` 里加载。

## 9. 什么时候看策略卡片

如果策略在产品线下有 `spec.yaml`，打分器会自动打印策略卡片。卡片信息应该固定记录：

- 策略 ID 和状态
- 假设和主仓位规则
- 调仓频率
- 风控模块
- 适用环境和弱势环境

这一步的意义是把“最终结论”从单文件代码里抽出来，避免后续多人在同一个 `strategy.py` 上直接回写，导致假设、参数和结论一起漂移。

## 10. 改完之后跑什么测试

```bash
env UV_CACHE_DIR=/tmp/uv-cache PYTHONPATH="$PWD" MPLCONFIGDIR=/tmp/mplconfig uv run pytest -q \
  tests/dividend_scorer \
  tests/unittest \
  tests/evaluation \
  tests/products
```

如果你只改了滚动打分器本身，至少跑 `tests/evaluation`。
如果你改了策略、mod 或评分器联动，建议直接跑上面这组基础测试。
如果你要跑 `tests/` 全量，先补齐 `TA-Lib`，否则部分 integration tests 会在收集阶段失败。
