# Regime Adaptive

这份策略是红利低波产品线针对“单边上涨吃不满”问题的独立试验版。

- 假设：价格趋势决定核心底仓，估值分位决定战术加减仓
- 结构：`strategy.py` 只保留 RQAlpha 适配层，真实逻辑拆到 `logic/`
- 参数：默认 profile 放在 `profiles/baseline.yaml`

后续如果只是调参数，新增 profile。
后续如果改变核心交易假设，继续新建策略目录，不回写 `history_aware`。

## 本轮结论

这一轮不是简单调参，而是把状态机真正接进了交易层：

- `v11` 把低分位恢复改成 `stage1/stage2`，并给高位热区补了状态化再入场路径。
- `v12` 修掉了一个执行层问题：状态机虽然给出了新目标，但 `deadband` 会把 `stage1/stage2` 的小幅恢复直接吞掉。
- 修完后，`2026-01-19` 的低位恢复首次真正成交，仓位从 `71% -> 76%`，不再停在 `71%` 左右空转。
- 同样的修复也让高位热区的分段再入场能真实下单，比如 `2025-07-21` 的 `18% -> 20%`、`2025-08-11` 的 `10% -> 18%`。
- 结果上，`v12` 的 `total_returns` 从 `6.7396%` 提升到 `6.8459%`，重新略微跑赢基准 `512890` 的 `6.8345%`。
- 月底净值差也按预期收敛：
  `2026-01` 稳定版相对差距从 `0.002347` 收窄到 `0.001355`，
  `2026-02` 从 `0.000196` 收窄到 `0.000023`。

这一轮新增/修正主要在：

- `logic/reentry_guard.py`
- `strategy.py`
- `tests/products/dividend_low_vol/strategies/regime_adaptive/test_logic.py`

## 现阶段还不行的点

现阶段它还不能替代稳定版，主要问题已经比较清楚：

1. `2025-07` 的高位热区仍然是最大失分段。
   当前稳定版相对 v12 的最大净值差，基本都集中在 `2025-07-09` 到 `2025-07-24`，说明热区里还是偏保守，单边上涨吃得不够满。

2. 低分位恢复虽然能成交了，但大多数时候只走到 `stage1`。
   `2026-01-19` 到 `2026-02-02` 这段基本停在 `76%`，`stage2` 没有真正放出来，所以恢复质量还是不如稳定版完整。

3. 换来更高收益的同时，交易噪音也上去了。
   `turnover` 从 `v11` 的 `3.1715` 增到 `v12` 的 `3.2885`，`information_ratio` 反而从 `1.7422` 回落到 `1.7066`，说明恢复路径还不够干净。

## 下阶段应该改什么

下一阶段不要推翻这套状态机，而是在此基础上继续细化：

1. 继续优化高位热区的状态化再入场。
   重点不是更早抄底，而是把 `cooldown/stage1/stage2` 和趋势强度绑定起来，让强趋势热区能更快恢复到更高的受控仓位。

2. 让低分位恢复的 `stage2` 更容易在“真修复”里被触发。
   可以从绝对阈值改成“相对改善 + 连续确认”，避免它长期卡在 `stage1=76%`。

3. 给状态机恢复路径单独加一层降噪。
   现在的 `deadband` 已经不能再粗暴拦它了，下一步应该考虑只对“状态维持周”做更细的微调过滤，别让成交次数继续抬高。

4. 如果这套状态机还是不够，再考虑继续拆 profile 做双轨实验。
   一个 profile 主攻收益恢复，一个 profile 主攻回撤约束，避免把单一 baseline 越调越糊。
